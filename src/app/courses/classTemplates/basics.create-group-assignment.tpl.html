What You Need To Know

Now for the final step of creating and using a group, assigning users to the user group. The typical first steps towards accessing any object as a buyer-user is to create the necessary objects and then assign them as necessary. This is the first of many create and assign combinations you will be executing. In fact, you will be creating three more objects and completing three more assignments in this course alone!

The reason for all of these assignments outside of the object creations themselves is to fit the profile of the modular nature of OrderCloud. It may seem like it is creating more steps for you to have to create and assign separately, but there is a reason for it. We want you to use exactly what you need to build an application for yourself or your clients, and nothing more. If we started creating nested objects directly inside of other objects (such as products inside of a category or prices inside of a product), it would require you to use more than you really need, thus slowing down the development time and vastly reducing your ability to define a proper project scope (this is the crux of a SAAS application; You have to use the whole package they offer). And as an added plus, you will find to love the modular orientation of OrderCloud once you get comfortable with it, because the separation of concerns makes it easier to manage and intuitively understand the structure of your environment.

Creating a Group Assignment

A full group assignment object is shown below,

Taking a look at the create.js script, you can see what is required to create a new Group assignment. Go ahead and replace the buyerID variable with the new buyer you created earlier in this course. In the assignment variable, a sample request object is currently stored with 3 values--UserGroupID, UserID, and BuyerID. Start by taking out the BuyerID object, as we won't need that (and it is already assumed in the parameter object).

You will be assigning two of your three new users to this group. You have two options to accomplish this. First you can process the create.js script twice with each of the users you are assigning. The second and more challenging option is to create a second assignment object (with a different variable name, of course) and process and second SDK call in the same script with that second assignment object.

If you are ready, go ahead and process the request by hitting "Run".
