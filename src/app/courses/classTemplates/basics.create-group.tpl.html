What You Need To Know

Creating a group is an easy and powerful way to give a subset of users the same access to a set of products, categories, pricing, permissions, and many other configurations that apply to a collection of users. They are easy to create and maintain, but most importantly, they are flexible. They manage to be flexible in two major ways. First, a user can be assigned to any amount of groups, whether that be 0, 1, or many. This means one user can have a combination of permissions from a mix of whatever groups they are assigned to. If group 1 has access to category A, and group 2 has access to category B, a user assigned to both gains access to both category A and B. Second, a group is given similar behaviors as a user. This means a group can be used in place of a user for nearly every assignment (category assignment, product assignment, address assignment, etc.).

What this flexibility means is that you can more easily create and scale an application with a massive or growing user base without sacrificing functionality due to administration getting out of hand. We urge you to create and use groups as often as you can, as it will help decrease the time it creates to build an administrative application, make it easier for the admin users to manage users, and even make it easier to allow buyer-users to self-administer.
