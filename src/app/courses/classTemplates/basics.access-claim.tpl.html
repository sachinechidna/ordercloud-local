What You Need To Know

API Access Claims are an important administrative service in OrderCloud. This service is where you will explicitly create each buyer application instance and where you will define the scope of each application's access rights. Without this you would not be able to authenticate as a buyer user, which would prove troubling within your buyer applications.

When processing a successful API Access Claim, an ID will be returned to you that will act as your applications clientID. For this class, we recommend that you let the ID auto-generate for this. This is what you will store in the base of your application as an app constant named clientid. This clientID is what the client application will use to tell OrderCloud what buyer environment it should be looking under when a user attempts to authenticate on the site.

Creating a New User

A full API Access Request Object is shown below. We will only be using part of this object, as the ClientSecret is not necessary for this exercise (we'll be using the administrative token stored in your browsers cookies for your current admin user), and DefaultUserContextID is an advanced topic we won't be touching on right now.

Review the documentation if you'd like a further review of each of these values. For now, we'll make a standard access claim and use that moving forward. First, go ahead and take out the ID and ClientSecret from the var claim object. We will let the ID auto-gen, and the ClientSecret is unnecessary for now. Set the AccessTokenDuration and RefreshTokenDuration to whatever you would like (keep in mind this is in milliseconds). Set the AppName as you please, and leave the Claims array as it is. Take out the DefaultUserContextID, and your var claim object is ready to go.

Finally, before you run the code block, you will need to set your var buyerID to the ID of the buyer you created in the Create a Buyer class. Also, please keep a note of the ID that is returned when you create this access claim, as you will be using it in the following class.

When you are ready, go ahead and hit 'Run'!
