What You Need To Know

A Buyer in OrderCloud is an organization or set of related users that submit orders to a distributor (wholesaler, manufacturer, etc.) through an OrderCloud application, having a similar level of access and using a similar set of purchasing rules. A Buyer is one of the foundations of an OrderCloud application, because it often provides the context in which some other object is being defined (such as a user or category).

Creating a Buyer is one of the more straightforward API Requests you will make in OrderCloud. In fact, you are only required to provide a name of your new buyer when creating one. If you choose to do this, your new buyer's ID will be auto-generated, and Active will be set to false. Creating a buyer uses the Buyers service provided from the SDK. The Buyers service, in turn, provides all the standard CRUD operations for dealing with Buyers.

Primary and secondary scripts will be standard in the How-To Courses, as we want to help you accomplish a specific goal, but also introduce you to the broader functionality and concepts that are specific to the class topic.

Now getting back to the topic at hand, creating a buyer. Over to the right in the create.js script, there are two pieces we want to focus on to successfully create a new Buyer.

First, on line 2 you can see a variable named buyer that stores the object of the new buyer you will be creating. All you need to do is set the ID and Name to whatever you please, and change Active to true. Note that the ID must not include any spaces (non-alphanumeric is okay). Additionally, it is important that you set Active to true, or you will run into issues later when trying to access the catalogue as a buyer-user. If you accidentally set it to false, you can go over to the API console and update the buyer object to make it active.

The second and final piece to create a new buyer is to make the SDK call to the OrderCloud API. This is a very simple SDK call, because it only requires one parameter--the buyer object. When you are ready, go ahead and run the script. We highly recommend that you make a note of your new buyer's ID so you can readily access and use it for future classes.
